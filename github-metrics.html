    Commented on #455 Unimplemented generator expr I found and seeming failure to build correct code for C++, go, or rust from py2many/py2many

    Now a bit more regarding the "generator exp" (node name GeneratorExp)- it is explicitly marked as unsupported in pycpp. See https://github.com/py2many/py2many/blob/b48f2fed0c9a0c43bb2bcae5e40824fd9e9bd472/pycpp/transpiler.py#L651

    This also means node types ListComp and SetComp are also unsupported in pycpp. Several languages dont support this yet.

    Regarding

    c = [ans.lower() for ans in c]

    The first problem is that str.lower isnt supported. I've created #504 about that.

    If we go with a simpler program like

    if __name__ == "__main__":
    a = ["A"]
(2 more lines were trimmed)

    This generates a .rs which is correct. You will need rustfmt installed, as that reformats the generated code and also validates that the generated code is sane.

    However go generates a mess.

    If we make it slightly simpler

    if __name__ == "__main__":
    a = ["A"]
(2 more lines were trimmed)

    Now Vlang and Julia also generate correct code.

    With regards to japanese_quiz.py , it currently uses

    • colorama which will not exist in any other language.
    • input which isnt supported - I've created #505 for this.

    As you can see, py2many has a long way to go, both in terms of usability, in support for basic Python language constructs, and in transpiler voodoo for the various target languages.
